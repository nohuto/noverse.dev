PnP Device Values

Switch to Zen Mode

This currently applies the values for the USB enumerator only, since most values were found in USB related drivers and kind of all of them (which I use in the option) only get read in the USB enumerator.

Disables USB selective suspend, idle states, and related LP features if supported.

Section titled “Registry Values”

Windows Plug and Play (PnP) creates a device node (devnode) for each detected device instance (“The PnP manager is the primary component involved in supporting the ability of Windows to recognize and adapt to changing hardware configurations.”). Common Subkeys under <instanceID>

Section titled “Common Subkeys under <instanceID>”
  • Device Parameters: Per-instance parameters and state used by the drivers in the stack
  • Properties: Device property store for this instance
  • LogConf (optional): Resource configuration data for the instance
  • Control (optional): Additional PnP/device state

Not every instance has the same subkeys or values. I won’t add details on the PnP manager here, as that’s not the purpose of the option. For more details, read Windows Internals E7, P1, Chapter 6 (The Plug and Play manager).

Default Data

Section titled “Default Data”

One thing to point out here is that there’re two APIs which I almost didn’t notice. IoOpenDeviceRegistryKey & PLUGPLAY_REGKEY_DEVICE opens the per-device-instance hardware key in the Enum branch (HKLM\SYSTEM\CCS\Enum\<Enumerator>\<DeviceID>\<InstanceID>\Device Parameters). IoOpenDriverRegistryKey opens the per-driver-service key in the Services branch (HKLM\SYSTEM\CCS\Services\<ServiceName>\Parameters). See power/assets/pnp for all used functions.

A simple example here would be GetEnhancedVerifierOptions which uses IoOpenDriverRegistryKey and as you can see in a boot trace, EnhancedVerifierOptions is used in for example \Registry\Machine\SYSTEM\ControlSet001\Services\PEAUTH\Parameters\Wdf : EnhancedVerifierOptions.

INF default = install-time default from INF entries.

To create this list, I’ve used many driver pseudocodes (usbhub, winhub, acpi, pci, wdf, hidclass, USBHUB3…), several INF files, and W10 source for comments (which may not be accurate anymore).

Everything listed below is based on personal findings, mistakes may exist.

Section titled “Storport Idle (Device Parameters\\StorPort)”
